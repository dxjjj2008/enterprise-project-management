## 1. 系统简介

### 1.1 系统概述
企业项目管理系统是一个集项目、任务、计划、资源、风险、问题于一体的综合管理平台，帮助团队高效协作、科学管理项目。

### 1.2 主要功能
- **项目管理**：创建、跟踪、管理项目
- **任务管理**：任务分配、进度跟踪、协作沟通
- **计划管理**：WBS分解、版本控制、基线对比
- **甘特图**：可视化时间轴、依赖关系
- **资源分配**：人员调度、工时管理
- **风险管理**：风险识别、应对跟踪
- **审批流程**：计划变更审批、交付确认
- **报表统计**：多维度数据报表

### 1.3 浏览器要求
推荐使用以下浏览器的最新版本：
- Google Chrome
- Mozilla Firefox
- Microsoft Edge

## 3. 项目管理

### 3.1 创建项目

1. 点击左侧导航"项目"
2. 点击右上角"新建项目"按钮
3. 填写项目信息：
   - **项目名称**：必填
   - **项目描述**：选填
   - **开始日期**：必填
   - **结束日期**：必填
   - **项目负责人**：必填
4. 点击"创建"完成

### 3.2 查看项目列表

![项目列表截图](/docs/screenshots/project-list.png)
*图：项目列表页面*

项目列表支持两种视图：
- **卡片视图**：以卡片形式展示项目
- **表格视图**：以表格形式展示项目列表

筛选和排序功能：
- 按状态筛选（全部/进行中/已完成/已归档）
- 按项目名称搜索
- 按最近更新时间排序

### 3.3 项目详情

点击项目卡片进入项目详情页，包含以下Tab：
- **概览**：项目基本信息、进度统计
- **任务**：任务列表
- **看板**：任务看板视图
- **甘特图**：项目时间线
- **文档**：项目文档管理
- **报表**：项目统计报表
- **成员**：项目团队成员

![项目详情截图](/docs/screenshots/project-detail.png)
*图：项目详情页面*

### 3.4 项目设置

在项目详情页点击"设置"按钮，可进行以下操作：
- 编辑项目信息
- 管理项目成员
- 配置项目标签
- 设置里程碑
- 项目归档

## 4. 任务管理

### 4.1 创建任务

**方式一：在项目任务列表创建**
1. 进入项目详情，点击"任务"Tab
2. 点击"新建任务"按钮
3. 填写任务信息：
   - **任务标题**：必填
   - **任务描述**：选填
   - **优先级**：高/中/低
   - **负责人**：选填
   - **截止日期**：选填
   - **所属里程碑**：选填
4. 点击"创建"完成

### 4.2 任务看板

![任务看板截图](/docs/screenshots/task-board.png)
*图：任务看板页面*

看板视图将任务分为四列：
- **Todo**：待办任务
- **In Progress**：进行中任务
- **In Review**：审核中任务
- **Done**：已完成任务

**拖拽操作**：
- 拖动任务卡片到其他列，更新任务状态
- 拖动调整任务顺序

### 4.3 任务详情

点击任务卡片可打开任务详情，包含：
- 任务基本信息（标题、描述、优先级、截止日期）
- 负责人信息
- 子任务列表
- 评论区域
- 附件区域
- 活动历史

![任务详情截图](/docs/screenshots/task-detail.png)
*图：任务详情弹窗*

### 4.4 子任务

在任务详情页可以创建子任务：
1. 在"子任务"区域点击"添加子任务"
2. 输入子任务标题
3. 按回车确认

### 4.5 任务筛选

在任务列表/看板页面，支持以下筛选：
- 按状态筛选
- 按优先级筛选
- 按负责人筛选
- 按里程碑筛选
- 按关键词搜索

## 5. 计划管理

### 5.1 WBS结构

WBS（Work Breakdown Structure）工作分解结构，将项目分解为可管理的任务单元。

**创建WBS任务**：
1. 进入项目详情，点击"计划"Tab
2. 点击"新建任务"按钮
3. 填写任务信息（支持设置层级关系）
4. 保存后自动形成WBS树形结构

### 5.2 版本管理

每次重大计划变更都会生成一个新版本：
1. 在计划页面点击"保存新版本"
2. 填写版本号和变更说明
3. 保存后可在版本历史中查看

### 5.3 基线对比

对比当前计划与基线版本的差异：
1. 在计划页面点击"版本对比"
2. 选择要对比的基线版本
3. 系统高亮显示：
   - 🟢 绿色：新增内容
   - 🔴 红色：删除内容
   - 🟡 黄色：修改内容

![计划管理截图](/docs/screenshots/plan-management.png)
*图：计划管理页面 - 版本对比*

### 5.4 提交审批

计划版本需要审批通过后才能生效：
1. 完成计划编制后，点击"提交审批"
2. 选择审批人
3. 填写审批说明
4. 等待审批结果

## 6. 甘特图

### 6.1 甘特图视图

![甘特图截图](/docs/screenshots/gantt-chart.png)
*图：甘特图页面*

甘特图以时间轴形式展示项目进度：
- 左侧：任务列表
- 右侧：时间轴和任务条

### 6.2 时间轴操作

- **缩放**：使用右上角按钮切换日/周/月视图
- **导航**：拖动时间轴或使用滚动条
- **今天定位**：点击"今天"按钮快速定位

### 6.3 任务条操作

- **查看详情**：点击任务条打开任务详情
- **调整日期**：拖动任务条左右边缘调整开始/结束日期
- **移动任务**：拖动任务条到新位置
- **调整工期**：拖动任务条宽度调整持续时间

### 6.4 里程碑

里程碑是项目的关键节点：
- 以菱形图标显示
- 无具体工期
- 可关联交付物

### 6.5 依赖关系

任务之间的依赖关系以箭头连接：
- **FS（开始-开始）**：前置任务开始后，当前任务才能开始
- **FF（完成-完成）**：前置任务完成后，当前任务才能完成
- **SF（开始-完成）**：前置任务开始后，当前任务才能完成

## 8. 风险管理

### 8.1 创建风险

1. 进入项目详情，点击"风险"Tab
2. 点击"新建风险"按钮
3. 填写风险信息：
   - **风险标题**：必填
   - **风险描述**：选填
   - **风险等级**：高/中/低
   - **发生概率**：高/中/低
   - **影响程度**：高/中/低
   - **应对措施**：选填
   - **责任人**：选填
4. 点击"创建"完成

### 8.2 风险应对

创建风险后，可以制定应对措施：
1. 点击风险卡片打开详情
2. 在"应对措施"区域添加措施
3. 设置负责人和完成时间
4. 跟踪应对进度

### 8.3 风险状态

风险状态流转：
- **已识别**：新录入的风险
- **已管控**：已制定应对措施
- **已发生**：风险已转化为问题
- **已关闭**：风险已解除

## 13. 常见问题

### Q1: 忘记密码怎么办？
A: 在登录页面点击"忘记密码"，通过邮箱验证重置密码。

### Q2: 如何添加项目成员？
A: 进入项目详情，点击"成员"Tab，点击"添加成员"，搜索并选择用户。

### Q3: 任务无法拖拽到其他列？
A: 请检查您是否有该任务的编辑权限。只有任务负责人或项目管理员可以拖拽任务。

### Q4: 审批被驳回怎么办？
A: 查看审批意见，根据意见修改后，可重新提交审批申请。

### Q5: 如何查看历史版本的计划？
A: 进入项目"计划"Tab，点击"版本历史"，选择要查看的版本。

### Q6: 资源分配超载如何处理？
A: 可考虑以下方案：
- 调整任务工期
- 增加资源（人员/设备）
- 调整任务优先级
- 将部分任务延期

### Q7: 甘特图不显示任务条？
A: 请检查任务是否设置了开始和结束日期，只有设置了日期的任务才会显示任务条。

## 14.4 阅读文档

### 14.4.1 文档查看器功能

系统提供专业的文档查看器，支持以下功能：

1. **固定顶部栏**
   - 顶部标题栏固定显示，不随页面滚动
   - 左侧显示返回按钮和当前文档名称
   - 右侧提供文档内容搜索框

2. **文档内搜索**
   - 在搜索框中输入关键词，实时高亮显示所有匹配内容
   - 自动跳转到第一个匹配位置
   - 搜索结果以黄色 `<mark>` 标签高亮显示

3. **左侧目录导航**
   - 自动从文档标题生成目录结构
   - 目录栏独立滚动，不影响正文阅读
   - 点击目录项可快速跳转到对应章节
   - 滚动时自动高亮当前可见章节

4. **独立滚动区域**
   - 左侧目录栏和右侧正文区域独立滚动
   - 保持标题栏始终可见
   - 提升长文档的阅读体验

### 14.4.2 操作说明

1. **选择文档**：在文档中心点击对应的文档卡片
2. **返回列表**：点击页面左上角"← 返回"按钮回到文档中心
3. **搜索内容**：在顶部搜索框输入关键词，按 Enter 搜索
4. **目录跳转**：点击左侧目录项跳转到对应章节
5. **滚动阅读**：左右区域独立滚动，互不影响

## 15. 无障碍性指南

### 15.1 基本原则

本系统遵循 WCAG 2.1 AA 标准，确保所有用户都能方便地使用系统，包括：
- 屏幕阅读器用户
- 键盘导航用户
- 低视力用户
- 色彩感知困难用户
- 其他有特殊需求的用户

### 15.2 键盘导航

系统支持完整键盘导航，无需鼠标即可完成所有操作。

#### 基本导航

| 快捷键 | 功能 | 说明 |
|--------|------|------|

### 15.3 焦点指示

系统使用蓝色边框清晰指示当前焦点位置。

- **按钮焦点**：蓝色边框 + 蓝色阴影
- **输入框焦点**：蓝色边框 + 边框发光效果
- **卡片焦点**：蓝色边框 + 上移效果 + 蓝色阴影
- **链接焦点**：蓝色下划线 + 蓝色阴影

#### 焦点顺序

焦点会按照以下顺序移动：
1. 主导航菜单
2. 全局搜索框
3. 页面主要内容
4. 侧边栏菜单
5. 页面底部链接
6. 页脚信息

### 15.4 屏幕阅读器支持

系统完全支持屏幕阅读器，所有重要内容都有清晰的描述。

#### 重要元素的屏幕阅读器描述

- **统计卡片**：读出卡片标题、数字、描述和趋势
  - 示例："进行中项目数量: 12个，较上周增加3个"

- **表单元素**：每个输入框都有明确的标签和错误提示
  - 示例："邮箱输入框，当前为空"

- **按钮**：所有按钮都有 aria-label 或明确文本
  - 示例："登录按钮，当前为"

- **链接**：所有链接都有明确的文本
  - 示例："注册链接"

- **进度条**：读出百分比和状态
  - 示例："完成率 57%"

#### ARIA 属性

系统自动添加 ARIA 属性，帮助屏幕阅读器理解页面结构：

- **role**：定义元素的语义角色
- **aria-label**：提供元素的描述性文本
- **aria-label**：为输入框提供额外的标签
- **aria-live**：为动态更新的区域提供支持
- **aria-hidden**：为装饰性图标隐藏屏幕阅读器输出

### 15.5 颜色对比度

系统遵循 WCAG 2.1 AA 标准，确保良好的色彩对比度：

- **正文文本**：≥ 4.5:1 对比度
- **UI 元素**：≥ 3:1 对比度
- **非文本内容**：≥ 3:1 对比度

#### 对比度标准

系统使用的颜色对比度：
- **深色文本** (#262626) vs 白色背景：约 10:1 ✅
- **中等文本** (#595959) vs 白色背景：约 7:1 ✅
- **浅色文本** (#595959) vs 灰色背景：约 7:1 ✅
- **功能色**：满足 WCAG 2.1 AA 标准 ✅

#### 对比度不足的颜色

以下颜色已优化：
- ~~#8C8C8C~~ → #595959 (对比度 7:1)
- ~~#BFBFBF~~ → #595959 (对比度 7:1)

### 15.8 常见问题

#### Q: 我可以使用键盘完成所有操作吗？

A: 是的，系统支持完整的键盘导航，所有功能都可以通过键盘完成。

#### Q: 我看不到焦点指示怎么办？

A: 请检查浏览器设置，确保焦点指示器没有被禁用。

#### Q: 快捷键冲突怎么办？

A: 您可以在系统设置中自定义快捷键绑定，避免冲突。

#### Q: 屏幕阅读器无法识别某些元素怎么办？

A: 请确保您使用的是最新版本的屏幕阅读器，并检查 ARIA 属性是否正确设置。

#### Q: 为什么有些颜色看起来不够清晰？

A: 系统已按照 WCAG 2.1 AA 标准优化颜色对比度，如仍有问题请告知我们。
